Important notes about the terminology of this documentation:

The word classifiers refer to the name of categorical variables associated with the dataset. For instance, yeah, color, etc... The word options are the levels associated with each classifier. For instance red, blue and green are options of color.

The word dependentVariable refers to the name of the field which holds the value to be plotted. Since this package assumes that you have data in a long rather than wide format (see data), there can only be one field per JSON cell which holds the value of interest (e.g., mean or median), and dependentVariable is the name which identifies this JSON field.

Another distinction that we make regards single versus multi classifiers. Multi-classifiers are those which the user chose to select more than one option. (e.g., year: 2010, 2011, color: Red, Blue). Single-classifiers are those which the user chose only one level (e.g., statistic: mean or median).

HTML structuring

Establish pre-defined divisions of the HTML page. Functions located at /src/generateHTMLstructure.js

initiateDashboard()

Generates structure of the HTML page.

# Parameters Description Default
1 renderMap [Boolean] Whether or not a canvas will be created for displaying interactive maps. false
2 directory [String] Path to the library's style sheets null

renderGraphBoxes()

Output: divs which will hold the main graphs.

# Parameters Description Default
1 nMultiClassifiers [Int] Number of multiclassifiers chosen by the user. value can be taken from the length of global variable multiClassifiers, which is the output of function pickMultiClassClassifiers() (see Category selectors section). none

generatePieChartsContainers()

Output: divs which will hold the pie-charts.

# Parameters Description Default
1 nPieCharts [Int] Number of pie-chart divs to be rendered on the HTML page. none

Category selectors

Family of functions that allow user to select different classifiers within the data set.

extractCategoriesAndOptions()

Output: list of two lists, where the first is a list of classifiers, and the second is a list of options extracted from the data. This output can be used as input parameter of function generateCheckBoxes (see below).

# Parameters Description Default
1 data [Object] JSON object. null
2 dependentVariableName [string] Key from JSON which names the field holding the value to be plotted. null

generateCheckBoxes()

Output: div with checkboxes for all classifiers and options. This allows users to select different options within each classifier. Subsequently, these choices will be used to filter the data (see function filterDataByCheckBox) and generate the graphs.

Depends on initiateDashboard().

# Parameters Description Default
1 classifiers [List] List of strings holding classifier names. null
2 options [List] List of lists of strings holding the options for each category. Order of the list needs to match the order of the category list. null
3 data [Object] JSON holding data to be plotted. null
4 dependentVariable [String] Object key indicating the JSON field which holds the values to be plotted. null
5 labels [Object] Object holding the labels to translate classifier and options codes into user-friendly names. For instance, translates the classifier rgn into Region. null
6 whereAppend [String] ID of the div which will hold the checkbox values. 'boxTop'

checkedValuesObjectGenerator()

Output: object with keys matching the structure of classifiers. Each key will indicate what options have been chosen by the user within each classifier. When initiated, the object will have only classifier keys, but empty values. The values of each keys will be updated every time a user makes a checkbox selection.

# Parameters Description Default
1 classifiers [List] List of classifiers. none

checkBoxVerificationSystem()

Output: system which verifies what values have been checked by the user. This updates the global variable generated with checkedValuesObjectGenerator(), and filters the dataset based on the user selection of checkboxes.

Function depends on the existance of a global value called checkedValues, which needs to be generated with the function checkedValuesObjectGenerator().

# Parameters Description Default
1 classifiers [List] List of classifiers. none
2 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null
3 data [Object] JSON holding data to be plotted. null
4 filterFunction [Function] Function which will read the selected check boxes and filter the data accordingly. We have a pre-defined version of this function (see filterDataByCheckBox). null
5 exception [String] Object key indicating one particular classifier that can only be selected as a single-classifier. null
6 howToFunction [Function] Function to execute when user presses how-to button. console.log("choose a function to execute here")

pickMultiClassClassifiers()

Output: verifies which classifiers are multi-classifiers (i.e., receive more than one options checked). Output is stored in a variable named multiClassClassifiers

# Parameters Description Default
1 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null
2 classifiers [List] List of classifiers. none

singleCheck()

Output: Automatically select one check box from a given classifier name. Checked option will be a random element within the option list for that classifier.

# Parameters Description Default
1 classifierName [String] String indicating tha name of one classifier to which a random option will be checked. null

multiCheck()

Output: Automatically select all check boxes from a given classifier.

# Parameters Description Default
1 classifierName [String] String indicating tha name of one classifier to which all options will be checked. null

simulateSelection()

Output: Automatically select checkboxes. There can be up to two multi-classifiers and any number of single-classifiers. Multi-classifiers will have all options checked, whereas single-classifiers will receive one check in a random option.

# Parameters Description Default
1 multi [List] List of strings containing the Object keys for the multi-classifiers. Length of the list is <= 2. null
2 single [List] List of strings containing the Object keys for the single-classifiers. null

verifyAllClassifiersChecked()

Output: Verifies if user chose at least one options for each classifier. If not, calls singleCheck() function with the name of that classifier as input.

# Parameters Description Default
3 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null

Data processing

Functions to facilitate the process of data for graphing.

filterDataByCheckBox()

Output: global variable named filteredData, which holds the filtered dataset based on selected check box option within each classifier.

# Parameters Description Default
1 classifiers [List] List of classifiers. none
2 data [Object] JSON holding data to be plotted. null
3 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null

separateDataInGroups()

Output: x-Axis and y-Axis to be used in graphing functions. The y-Axis will be separated beased on the options chosen for the multi-classifier.

# Parameters Description Default
1 filteredData [Object] Object stored within the global variable filteredData, which is generated as the output of checkBoxVerificationSystem(). Filtering is done based on user selection of check boxes. null
2 groupSelected [String] Classifier which has been selected as a multi-clssifier. This variable can be taken from the output of the function pickMultiClassClassifiers(). null
3 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null

Graphing

Enable the rendering of different types of graphs. The base dependency of these functions is the ChartJS library.

colorGenerator()

Output: list of colors. One for each data group (defined by multi-classifiers).

# Parameters Description Default
1 yAxis [List] List of lists containing values to be plotted. Output of function separateDataInGroups(). none

graphCustom()

Output: chart generated by library ChartJS and displayed on the HTML document. Depends on the functions initiateDashboard() and renderGraphBoxes(), which will hold the divs where charts will be displayed.

# Parameters Description Default
1 xAxis [List] List of strings or numbers. Output of function separateDataInGroups() none
2 yAxis [List] List of lists containing values to be plotted. Output of function separateDataInGroups(). none
3 labels [List] List of options chosen by user for a given multi-classifier. Values can be extracted from the output of function separateDataInGroups(). none
4 id [String] ID of canvas tag where chart will be rendered. If initiateDashboard() and renderGraphBoxes() are used, value must be "myChart" or "myChart1" none
5 title [String] Type of graph to be rendered. Either 'line' or 'bar' none
6 showLegend [Boolean] true or false indicating wether legend of the graph should be displayed. true
7 fill [Boolean] true or false indicating wether are below line should be filled with color. false
8 suggestedMin [Int] Number indicating the suggested minimum value of the y-axis. null
9 position [String] Position of legend. Either 'bottom', 'top', 'left' or 'right'. bottom

graphCustomPie()

Output: chart generated by library ChartJS and displayed on the HTML document. Depends on the functions initiateDashboard() and renderGraphBoxes(), which will hold the divs where charts will be displayed.

# Parameters Description Default
1 xAxis [List] List of strings or numbers. Output of function separateDataInGroups() none
2 yAxis [List] List of lists containing values to be plotted. Output of function separateDataInGroups(). none
3 labels [List] List of options chosen by user for a given multi-classifier. Values can be extracted from the output of function separateDataInGroups(). none
4 id [String] ID of canvas tag where chart will be rendered. If initiateDashboard() and renderGraphBoxes() are used, value must be "myChart2", "myChart3", or "myChart4". none
5 title [String] Type of graph to be rendered. Either 'line' or 'bar' none
6 showLegend [Boolean] true or false indicating wether legend of the graph should be displayed. true
7 fill [Boolean] true or false indicating wether are below line should be filled with color. false
8 suggestedMin [Int] Number indicating the suggested minimum value of the y-axis. null
9 position [String] Position of legend. Either 'bottom', 'top', 'left' or 'right'. bottom

displayNonGraphs()

Output: styled div in the middle of the HTML document. This div will appear every time the filtered data finds no data, or is entirelly composed of null values.

# Parameters Description Default
1 filteredData [Object] Object stored within the global variable filteredData, which is generated as the output of checkBoxVerificationSystem(). Filtering is done based on user selection of check boxes. none

displaySelectedSingleVariables()

Output: Display selected single-classifiers. Result will be appended to the HTML page, and user will be able to see which single-variables are being used for filtering the data. Selected multi-classifiers are already visible in the x-axis and in labels of the graphs.

# Parameters Description Default
1 checkedValues [Object] Global object generated with the function checkedValuesObjectGenerator(). null
2 exception [String] Object key which does not have a label translation. Only usefull if label is passed as an argument. null
3 labels [Object] Object holding the labels to translate classifier and options codes into user-friendly names. For instance, translates the classifier rgn into Region. null
